# Code of Conduct

## Protocol Alignment (Normative)

SPEX means **Secure Permissioned Exchange**.
SPEX is a **protocol**, not just an application.
Security comes before convenience.
Core cryptographic invariants are non-negotiable.
All architecture and behavior described in this document must remain aligned with:
**Secure. Permissioned. Explicit.**

## Our Pledge

We pledge to make participation in the SPEX project a respectful,
professional, and technically focused experience for everyone.

SPEX is a security-sensitive project. Discussions should prioritize
clarity, correctness, and evidence over opinion or ego.

---

## Expected Behavior

Participants are expected to:

- be respectful and professional
- focus discussions on technical merit
- accept constructive criticism
- disagree without hostility
- assume good intent unless proven otherwise

Technical rigor is encouraged.
Personal attacks are not.

---

## Unacceptable Behavior

The following behaviors are not acceptable:

- harassment or discrimination of any kind
- personal attacks or insults
- trolling, baiting, or deliberate disruption
- dismissing security concerns without analysis
- pressuring maintainers to weaken security guarantees

---

## Scope

This Code of Conduct applies to all project spaces, including:

- GitHub issues
- pull requests
- discussions
- code reviews
- documentation comments

---

## Enforcement

Project maintainers are responsible for enforcing this Code of Conduct.

They may take any action deemed appropriate, including:

- requesting changes in behavior
- closing issues or pull requests
- temporarily or permanently banning contributors

Decisions are made to protect the project and its users.

---

## Reporting

If you experience or witness unacceptable behavior, please contact
the maintainers privately.

For security-related issues, follow the process described in
`SECURITY.md`.

---

## Final Note

SPEX values technical excellence and respectful collaboration.

Strong opinions are welcome.
Disrespect is not.
